About RC Car

A Remote-Control (RC) car is a miniature model car that can be controlled from a distance using a specialized transmitter. In this project, we utilize an Arduino UNO as the brain of the vehicle, combined with a Bluetooth module to enable seamless control via a smartphone app. This project demonstrates fundamental concepts of embedded systems, wireless communication, and motor control.

Applications

  • Educational Robotics
  • Search and Rescue Prototypes
  • Entertainment and Hobby
  • Prototyping Autonomous Vehicles

Components Used

Arduino UNO

The Brain

Features: ATmega328P, 14 Digital I/O, 6 Analog inputs.

Working: Processes incoming Bluetooth signals and sends appropriate signals to the motor driver.

L298N Motor Driver

Motor Controller

Features: Dual H-Bridge, handles up to 2A per channel.

Working: Allows speed and direction control of two DC motors independently based on Arduino signals.

HC-05 Bluetooth Module

Wireless Receiver

Features: SPP protocol, range ~10m.

Working: Receives serial data from the mobile app and transmits it to the Arduino's RX pin.

DC Gear Motors

Actuators

Features: High torque, low speed.

Working: Converts electrical energy into mechanical rotation to drive the wheels.

Servo Motor

Steering Mechanism

Features: Precise angle control (0-180 degrees).

Working: Steers the front wheels accurately left or right based on PWM signals.

Ultrasonic Sensor

Obstacle Detection

Features: Non-contact distance measurement (2cm - 400cm).

Working: Emits sound waves and measures echo time to calculate distance to obstacles.

Source Code

main.ino 

// RC Car Project - Bluetooth Control
#include <Servo.h>

// Motor A (Right)
int enA = 9;
int in1 = 8;
int in2 = 7;
// Motor B (Left)
int enB = 3;
int in3 = 5;
int in4 = 4;

char command;

void setup() {
  Serial.begin(9600); // Initialize Bluetooth communication
  
  // Set all motor control pins to outputs
  pinMode(enA, OUTPUT);
  pinMode(enB, OUTPUT);
  pinMode(in1, OUTPUT);
  pinMode(in2, OUTPUT);
  pinMode(in3, OUTPUT);
  pinMode(in4, OUTPUT);
}

void loop() {
  if (Serial.available() > 0) {
    command = Serial.read();
    Stop(); // Initialize with motors stopped
    
    switch(command) {
      case 'F': Forward(); break;
      case 'B': Reverse(); break;
      case 'L': Left(); break;
      case 'R': Right(); break;
    }
  }
}

void Forward() {
  analogWrite(enA, 255);
  analogWrite(enB, 255);
  digitalWrite(in1, HIGH);
  digitalWrite(in2, LOW);
  digitalWrite(in3, HIGH);
  digitalWrite(in4, LOW);
}

void Reverse() {
  analogWrite(enA, 255);
  analogWrite(enB, 255);
  digitalWrite(in1, LOW);
  digitalWrite(in2, HIGH);
  digitalWrite(in3, LOW);
  digitalWrite(in4, HIGH);
}

void Left() {
  analogWrite(enA, 255);
  analogWrite(enB, 255);
  digitalWrite(in1, HIGH);
  digitalWrite(in2, LOW);
  digitalWrite(in3, LOW);
  digitalWrite(in4, HIGH);
}

void Right() {
  analogWrite(enA, 255);
  analogWrite(enB, 255);
  digitalWrite(in1, LOW);
  digitalWrite(in2, HIGH);
  digitalWrite(in3, HIGH);
  digitalWrite(in4, LOW);
}

void Stop() {
  digitalWrite(in1, LOW);
  digitalWrite(in2, LOW);
  digitalWrite(in3, LOW);
  digitalWrite(in4, LOW);
}

Code Explanation

  • Setup: Initializes the serial communication for the HC-05 module and sets motor pins as outputs.
  • Loop: Continuously checks for incoming Bluetooth data. Reads the command and executes corresponding movement functions.
  • Movement Functions (Forward, Reverse, Left, Right): Controls the H-Bridge logic (IN1-IN4) and PWM speed (enA/enB) to move the car in the desired direction.

Working Process

1

Mobile App

2

HC-05 Receiver

HC-05 module receives commands via SPP.

3

Arduino Processing

Arduino parses commands from serial buffer.

4

Motor Control

L298N driver directs power to DC motors based on logic pins.

5

Steering

Servo handles precise steering angles (if using Ackermann steering).

6

Safety

Ultrasonic sensor detects obstacles and stops motors if path is blocked.

Frequently Asked Questions

What app can I use to control the car?

You can use any standard "Arduino Bluetooth Car" app available on the Google Play Store, or build your own using MIT App Inventor.

How long does the battery last?

With standard 18650 Li-ion cells (approx 2000mAh), the car can run for about 1-2 hours continuously depending on motor load.

Can I add a camera to the car?

Yes, you can upgrade the project by adding an ESP32-CAM module to stream live video to your device.
